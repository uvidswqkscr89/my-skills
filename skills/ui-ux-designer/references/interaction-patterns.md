# Interaction Patterns | 交互模式参考

常用交互模式的设计规范，供 ui-ux-designer skill 使用。

---

## 1. 自动滚动（Auto-scroll）

**场景**：页面加载后自动定位到关键内容（如失败步骤）

**规范**：
- 触发时机：页面/组件完成渲染后 300ms
- 滚动行为：`scrollIntoView({ behavior: 'smooth', block: 'center' })`
- 视觉提示：目标元素短暂高亮（300ms 渐隐）
- 边界处理：若目标元素不存在，不触发滚动

---

## 2. 展开/折叠（Expand / Collapse）

**场景**：错误详情、高级选项、长文本

**规范**：
- 默认状态：根据内容重要性决定（失败详情默认展开）
- 动画：高度从 0 到 auto，200ms ease-in-out
- 图标：`▼` 展开，`▲` 折叠，或 `+` / `-`
- 点击区域：整行可点击（不只是图标）

---

## 3. 静态预览 + 播放（Preview + Play）

**场景**：截图预览，点击后播放录像

**规范**：
- 默认展示：静态图片，右下角显示 `▶` 播放图标覆盖层
- Hover 状态：播放图标放大，图片轻微变暗（overlay opacity 0.3）
- 点击行为：进入播放模式，从第一帧开始
- 说明文字：图片下方，`text.body.sm`，`color.text.secondary`
- 加载失败降级：显示占位图 + "截图加载失败"文字

---

## 4. 内联操作链接（Inline Action Link）

**场景**：步骤行内的"查看录像"等快捷操作

**规范**：
- 位置：行内右对齐，或错误详情区底部
- 样式：文字链接，`color.info`，无下划线，hover 时显示下划线
- 仅在相关状态下显示（如仅失败步骤显示"查看录像"）
- 点击：新标签页打开，或页面内跳转

---

## 5. 状态感知渲染（Conditional Rendering by Status）

**场景**：根据 Run 状态（failed/success/running）渲染不同 UI

**规范**：
```
if status === 'failed':
  右侧默认展示最后截图（静态预览）
  失败步骤自动展开错误详情
  自动滚动到失败步骤
else if status === 'success':
  右侧保持现有行为（从第一帧播放）
  步骤列表正常展示
else if status === 'running':
  右侧展示实时截图流或加载态
  步骤列表实时更新
```

---

## 6. 降级处理（Graceful Degradation）

**原则**：数据缺失不应阻断主流程，只影响对应的增强功能。

| 缺失数据 | 降级处理 |
|----------|----------|
| `error_summary` 为空 | 显示 "No error details available"，样式保持 |
| `failed_step` 为 null | 不显示步骤定位文字，其他内容正常 |
| 最后截图加载失败 | 显示占位图 + 说明文字，保留"播放录像"入口 |
| `steps[]` 为空 | Create Bug 描述中跳过步骤列表区块 |

---

## 7. 表单预填充（Pre-filled Form）

**场景**：Create Bug 弹窗自动填入内容

**规范**：
- 自动填充字段：描述（步骤列表 + 失败原因 + 录像 + 截图）
- 用户可编辑：所有预填内容均可修改
- 视觉提示：预填内容区域有轻微背景色区分，或顶部提示"已自动填入测试上下文"
- 字数限制：若描述超出 bug 系统字数限制，截断并提示

---

## 8. Toast / 通知（Feedback Notification）

**场景**：操作成功/失败的即时反馈

**规范**：
- 位置：页面右上角
- 持续时间：成功 3s，错误 5s（或手动关闭）
- 类型：success（绿）、error（红）、warning（橙）、info（蓝）
- 内容：简短动词短语，如"Bug 已创建"、"截图加载失败，已降级展示"
